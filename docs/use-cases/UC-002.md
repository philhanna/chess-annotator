# UC-2: List Games

**Actor:** Author

**Preconditions:**
- The store root is configured and accessible

**Main Flow:**
1. Author requests the list of games
2. System scans the store and returns one entry per game directory
3. Each entry shows the game-id, player names, event, date, result, and whether a session is in progress

**Alternate Flows:**
- **Store is empty** — system returns an empty list with no error

**Postconditions:**
- No state is changed
