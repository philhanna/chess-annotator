# UC-9: Set Segment Annotation

**Actor:** Author

**Preconditions:**
- A session is open for the game
- The segment exists

**Main Flow:**
1. Author selects a segment
2. Author supplies annotation text
3. System updates the segment's annotation text in the working annotation JSON
4. System returns the updated segment

**Alternate Flows:**
- **segment not found** — system reports the error; nothing is changed
- **annotation is blank** — system reports the error; nothing is changed

**Postconditions:**
- The segment has the new annotation text
- The session remains open with unsaved changes
