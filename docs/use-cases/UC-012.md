# UC-12: Close Game

**Actor:** Author

**Preconditions:**
- A session is open for the game

**Main Flow:**
1. Author requests close
2. System checks whether the working files differ from the main files
3. If there are no unsaved changes, system deletes the working files
4. If there are unsaved changes, system asks whether to save or discard
5. If author chooses save, system saves the working files to the main files and then deletes the working files
6. If author chooses discard, system deletes the working files without updating the main files

**Alternate Flows:**
- **author cancels at the save prompt** — system leaves the session open and changes nothing

**Postconditions:**
- The session is closed
- No `.work` files remain for the game
