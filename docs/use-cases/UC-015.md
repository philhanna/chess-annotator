# UC-15: List Segments

**Actor:** Author

**Preconditions:**
- A session is open for the game

**Main Flow:**
1. Author requests the segment list
2. System loads the working annotated PGN and working annotation JSON
3. System derives the segments from the turning points
4. System returns one entry per segment showing the move range, label, and annotation status

**Alternate Flows:**
- **session is not open** — system reports the error and returns no segment list

**Postconditions:**
- No state is changed
