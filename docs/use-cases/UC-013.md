# UC-13: Render PDF

**Actor:** Author

**Preconditions:**
- The game exists in the store
- Every segment has both a label and an annotation

**Main Flow:**
1. Author requests PDF rendering for a game
2. System loads the current annotation state from the working files if a session is open, otherwise from the main files
3. System derives the segments, renders diagrams for segments whose `show_diagram` flag is true, and assembles the HTML document
4. System renders the HTML to PDF
5. System writes `output.pdf` in the game directory and returns its location

**Alternate Flows:**
- **game-id not found** — system reports the error; nothing is rendered
- **a segment is missing a label or annotation** — system reports the error; no PDF is created
- **PDF generation fails** — system reports the error; any existing `output.pdf` is left unchanged

**Postconditions:**
- `output.pdf` contains the rendered annotation for the current game state
