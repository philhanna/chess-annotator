# UC-6: Add Turning Point

**Actor:** Author

**Preconditions:**
- A session is open for the game

**Main Flow:**
1. Author supplies a ply within the game
2. System validates that the ply is within range and is not already a turning point
3. System adds a `[%tp]` marker at that ply in the working annotated PGN
4. System creates a new segment entry in the working annotation JSON for that turning point
5. System derives the updated segment list and returns it

**Alternate Flows:**
- **ply is outside the game** — system reports the error; nothing is changed
- **ply is already a turning point** — system reports the error; nothing is changed

**Postconditions:**
- The current segment is split into two segments at the requested ply
- The session remains open with unsaved changes
