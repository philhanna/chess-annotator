# UC-4: Save Game As

**Actor:** Author

**Preconditions:**
- The source game exists in the store
- The store root is configured and accessible

**Main Flow:**
1. Author supplies a source game-id and a new game-id
2. System checks whether the new game-id is already in use
3. System creates the new game directory
4. If the source game has an open session, system copies `annotated.pgn.work` and `annotation.json.work` into the new game's main files
5. If the source game has no open session, system copies `annotated.pgn` and `annotation.json` into the new game's main files
6. System registers the new game under the new game-id without changing the source game

**Alternate Flows:**
- **source game-id not found** — system reports the error; nothing is created
- **new game-id already exists** — system warns and asks to confirm overwrite or cancel
- **overwrite confirmed** — system replaces the target game's files with copies from the source game

**Postconditions:**
- A new game exists in the store with the same current PGN and annotation state as the source game
- The source game is unchanged
- No session is opened for the new game
