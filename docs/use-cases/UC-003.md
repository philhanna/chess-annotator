# UC-3: Open Game

**Actor:** Author

**Preconditions:**
- The game exists in the store

**Main Flow:**
1. Author supplies a game-id
2. System locates the game directory and loads the annotated PGN and annotation JSON
3. If no `.work` files exist, system creates them from the main files and opens a new session
4. If `.work` files already exist, system loads them as-is (resuming a previous session)
5. System returns the game's current state: metadata, segment list, and session status

**Alternate Flows:**
- **game-id not found** — system reports the error; no session is opened

**Postconditions:**
- A session is open for the game
- If the session is new, `.work` files are identical to the main files
- If the session is resumed, `.work` files are unchanged
